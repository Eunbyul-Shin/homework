# 마크다운 문법

## 마크다운 정의

텍스트 기반의 마크업 언어로 2004년 존그루버로 인해 쉽계 읽을 수 있으며, HTML로 변환이 가능하다. 정보를 기록하는 Readme.md는 깃허브를 사용하는 사람이라면 가장 먼저 시작할 수 있는 마크다운 문서로 마크다운을 사용할 수 있는 설치 방법, 소스 코드 설명, 문제 등을 간단하게 기록하고 가독성을 가질 수 있다.

장점은 간결하고 별도의 도구 없이 작성 가능하다는 점이다. txt형태로 저장되기 때문에 용량이 적다.

단점으로는 표준이 없다. 그래서 도구에 따라서 변환방식이나 생성물이 다를 수 있다. 모든 HTML 마크업을 대신하지 못 한다.

## 아래로는 마크다운 문법에 대해서.

1. 헤더 태그로 웹 페이지 콘텐츠의 가독성을 위해 사용된다. H7은 존재하지 않는다.

# This is a H1

## This is a H2

### This is a H3

#### This is a H4

##### This is a H5

###### This is a H6

2. 이메일에 사용하는 블록인용 문자를 사용하여 인용 블록을 만든다. 해당 문법에는 다른 마크다운 요소를 중복으로 사용할 수 있다.(ex.헤더 태그)

> This is a first blockqute.
> This is a second blockqute.
>
> > This is a third blockqute.

3. 목록은 숫자와 점을 사용하여 나타낸다.

1. 첫번째
1. 두번째
1. 세번째

1. 외부 목록은 글머리 기호를 지원한다.

- 빨강
  - 녹색
    - 파랑

* 빨강
  - 녹색
    - 파랑

- 빨강
  - 녹색

## 아래로는 마크다운 코드에 대해서.

4개의 공백 또는 하나의 탭으로 프레스 쓰기를 만나면 변환되기 시작하여 프레스를 쓰지 않은 배열을 만들 때까지 변환이 계속된다.

코드블럭은 다음과 같이 2가지 방식으로 사용할 수 있다:

1. <pre><code>{code}</code></pre>이용방식
2. 코드블럭코드("```") 이용방식
   깃허브 내에서 시작점에 사용하는 언어를 선언하여 구문 강조가 가능하다.
3. <hr/>
